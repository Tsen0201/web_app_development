# PRD: 任務管理系統

## 1. 專案概述
- **背景與動機**：為了有效管理個人生活與工作中的各項待辦事項，避免遺漏重要事項並確保進度，需要一款能夠新增、刪除並透過日曆檢視的任務系統，同時具備提醒能力。
- **目標用戶**：我自己（單一個人用戶）。
- **核心價值主張**：提供簡單、直觀且包含月曆檢視與提醒功能的個人管理工具，以提高排程與做事效率。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **新增與刪除任務**
   - 作為我自己，我希望可以新增與刪除任務，以便隨時更新目前的待辦清單並移除不需要的項目。
2. **詳細描述任務內容**
   - 作為我自己，我希望可以詳細描述每個任務的內容與細節，以便未來執行時能清楚記得任務的脈絡與要求。
3. **任務時間設定與日曆檢視**
   - 作為我自己，我希望可以設定每個任務的時間，並且能在一個日曆畫面上看到所有的任務分布，以便直觀地規劃並檢視我的每日/每週行程。
4. **任務狀態與完成確認**
   - 作為我自己，我希望可以標記並確認任務已經完成，以便追蹤整體進度並獲得達成目標的成就感。
5. **任務分類**
   - 作為我自己，我希望可以將任務進行分類（例如：工作、日常、休閒等），以便更有條理地篩選與管理不同面向的待辦事項。
6. **任務提醒**
   - 作為我自己，我希望能在任務設定的時間到時或當天收到提醒，以便不會忘記處理重要任務。
7. **週期性重複任務**
   - 作為我自己，我希望可以設定每一段時間（例如每日、每週或每月）自動重複任務，以便管理例行性的代辦事項，節省重複建立的時間。

## 3. 非功能需求
- **技術限制**
  - 使用框架：後端使用 Python Flask，前端渲染使用 Jinja2 Template，搭配原生 HTML/CSS/JS。
  - 資料庫：使用 SQLite 做為儲存引擎。
- **效能考量**
  - 使用者數量極少（單人），因此重點在於應用程式的載入速度需合理即可。
  - 日曆呈現資料時需確認查詢效率是否平順。
- **安全考量**
  - 作為單機或個人使用的系統，沒有複雜的權限區分，若要部署至公網則需考慮基本的帳號密碼保護防止個人資料被他人存取。

## 4. MVP 範圍
- **Must Have (必須具備)**
  - 新增、修改、刪除任務功能。
  - 詳細記錄任務內容，並能勾選完成。
  - 任務的時間設定與月曆視圖檢視。
  - 任務的基本分類管理功能。
- **Should Have (應該具備)**
  - 任務到達設定時間或當日開啟時的首頁/通知提醒。
- **Nice to Have (有了更好)**
  - 週期性重複任務（例行性代辦）的自動產生功能。
